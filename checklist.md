# AI Chatbox Web MVP 验收标准

## 1. 文档与规划验收

- [x] 已形成明确的项目需求大纲，覆盖目标、范围与技术方案
- [x] 已创建 `spec.md` 作为项目开发总纲
- [x] 已创建 `tasks.md` 作为任务推进清单
- [x] 已创建 `checklist.md` 作为验收核对清单

## 2. 功能验收

- [ ] 用户可以在页面中输入并发送消息
- [ ] 助手可以返回至少一轮有效回复
- [ ] 支持多轮上下文对话
- [ ] 支持创建多个会话并在会话之间切换
- [ ] 刷新页面后历史会话仍可恢复
- [ ] 支持 mock 测试模式完成完整聊天流程
- [ ] 支持 OpenAI 兼容接口模式完成完整聊天流程

## 3. 交互与界面验收

- [ ] 用户消息发送后可立即显示在消息列表中
- [ ] 助手回复以流式或分段方式增量显示
- [ ] 输入框支持回车发送和 Shift+Enter 换行
- [ ] 发送过程中不会重复提交同一请求
- [ ] 错误场景下页面能显示明确失败提示
- [ ] 页面在桌面端和移动端均可正常使用

## 4. 接口与安全验收

- [ ] 真实 API Key 仅存在于服务端环境变量中
- [ ] `POST /api/test/chat` 需要 demo key 才可访问
- [ ] `GET /api/health` 能返回当前 provider 和模型状态
- [ ] 请求体异常时接口能返回可识别的错误信息

## 5. 工程与质量验收

- [ ] 项目依赖可以正确安装
- [ ] 项目可以在本地成功启动
- [ ] 关键单元测试通过
- [ ] 核心 E2E 测试通过
- [ ] 代码结构清晰，具备继续扩展能力
- [ ] `README.md` 足以指导其他开发者完成本地运行与联调

## 6. 交付验收

- [ ] 已完成本期 MVP 范围内的核心开发任务
- [ ] `tasks.md` 中已完成的开发项均已同步标记
- [ ] `checklist.md` 中满足条件的验收项均已同步标记
- [ ] 项目具备进入下一阶段迭代的基础
